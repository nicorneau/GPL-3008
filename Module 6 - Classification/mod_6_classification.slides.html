6.3 Régression logistique¶

  • Dans le cas o la variable dpendante prend seulement deux valeurs, il est possible d'crire :
$$y_i = \bigg\{ \begin{array}{c c} 1 \\ 0 \end{array}$$
  • $y_i$ suit donc une loi Bernouilli :
$$\Pr(y_i) = \pi_i^{y_i}(1-\pi_i)^{(1 - y_i)}$$

o $\pi_i = \Pr(y_i = 1)$.

  • Il a t montr prcdemment que dans le cas du modle de probabilit linaire, $\pi_i$ se modlise ainsi
$$\Pr(y_i = 1| x_i) = \pi_i = x_i'\beta$$
  • Il a galement t dmontr que ce modle est problmatique puisque $\hat{\pi}_i$, c'est--dire la prdiction du modle, n'est pas borne entre zro et un.
  • La rgression logistique impose une structure entre $\pi_i$ et $x_i'\beta$ de faon ce que la probabilit $\pi_i$ soit bien dfinie entre zro et un.
  • Formellement, la rgression logistique impose
$$\pi_i = \frac{\exp\{x_i'\beta\}}{1 + \exp\{x_i'\beta\}}$$
  • Comme il est possible de le voir, si $x_i'\beta \rightarrow \infty$, alors $\pi_i \rightarrow 1$ tandis que si $x_i'\beta \rightarrow -\infty$, alors $\pi_i \rightarrow 0$.
  • Cette structure est parfois crite de faon alternative
$$\pi_i = \text{Logit}^{-1}(x_i'\beta) = \Lambda(x'_i\beta)$$
  • En effet, aprs quelques manipulations simples, il est possible de montrer que
$$x'\beta = \text{Logit}(\pi_i) = \ln\frac{\pi_i}{1 - \pi_i}$$
  • Ainsi, la distribution de $y$ peut tre rcrite :
$$\Pr(y_i| x_i) = \Lambda(x'_i\beta)^{y_i}\left(1-\Lambda(x'_i\beta)\right)^{(1 - y_i)}$$

Estimateur du maximum de vraisemblance

  • La thorie des probabilits stipule que si deux vnements sont indpendants, alors leur probabilit jointe est simplement le produit de leur probabilit marginale :
$$\Pr(y_i, x_i) = \Pr(y_i)\Pr(x_i)$$
  • L'estimation du maximum de vraisemblance (maximum likelihood estimator, MLE) repose sur l'hypothse que les observations composant l'chantillon ont t slectionnes dans une mme distribution de faon alatoire et donc indpendamment les unes des autres.
  • Ainsi, leur probabilit jointe est simplement le produit de leur probabilit individuelle :
$$\Pr(y_1, y_2, ..., y_n) = \Pr(y_1)\Pr(y_2),...,\Pr(y_n)$$

o $\Pr(y_i)$ est la probabilit que l'observation $i$ soit observe. Ainsi, $\Pr(y_1, y_2, ..., y_n)$ reprsente la probabilit d'observer l'chantillon compos de $n$ observations.

  • La fonction reprsentant $\Pr(.)$ dpend gnralement d'un ou de plusieurs paramtres $\theta$. Le produit ci-haut peut donc tre rcrit tel que
$$\Pr(y_1, y_2, ..., y_n; \theta) = \Pr(y_1; \theta)\Pr(y_2; \theta),...,\Pr(y_n; \theta)$$

o $\theta$ est un vecteur de paramtres.

  • Puisque l'hypothse postule que les observations proviennent de la mme distribution, $\theta$ est le mme pour toutes les observations.
  • Le vecteur de paramtres $\theta$ est gnralement inconnu. Il est alors possible d'crire une fonction dfinissant comment le produit des probabilits ci-haut varie selon la valeur des paramtres $\theta$ :
$$L(\theta) = \prod_{i=1}^N\Pr(y_i; \theta)$$
  • Cette expression est connue sous le nom de fonction de vraisemblance (likelihood function).
  • La vraisemblance est une fonction reprsentant la probabilit que l'chantillon disponible soit observ pour diffrentes valeurs de $\theta$.

Exemple :

  • Soit une variable $age = \{42, 24, 36, 29, 51\}$ provenant d'une distribution normale.
  • Un analyste est intress estimer la moyenne de la population partir de cet chantillon.
  • Soit deux valeurs candidates pour la moyenne de la variable age : $\theta = \{10, 37\}$.
  • Quiz : Pour l'analyste, quelle valeur de $\theta$ est la plus probable, sachant l'chantillon qu'il observe?
  • La fonction de vraisemblance repose sur cette intuition.
  • Elle retourne diffrentes valeurs pour diffrents $\theta$ candidats. La plus grande valeur qu'elle retourne (le maximum de la fonction de vraisemblance) se trouve au $\theta$ le plus probable sachant l'chantillon observ.
  • Trouver le maximum de la fonction est alors simplement un problme d'optimisation.
  • Puisque la fonction de vraisemblance est le rsultat du produit des probabilits individuelles, prendre le logarithme de celle-ci permet d'obtenir la fonction suivante :
$$\text{ln} L(\theta) = l(\theta) = \sum_{i=1}^n\text{ln}\left[\Pr(y_i;\theta)\right]$$
  • Cette transformation est souhaitable puisqu'elle permet de mieux faire converger le modle.
  • Il est possible de montrer que cette transformation ne change pas la valeur du maximum de la fonction.
  • Il est noter que l'estimateur du maximum de vraisemblance est l'un des estimateurs les plus rpandus en statistique, car il possde des proprits statistiques trs intressantes.
  • Le vecteur de paramtres $\theta$ est alors obtenu en solutionnant le problme de maximisation suivant :
$$\max_{\theta} ~ l(\theta) = \sum_{i=1}^n\text{ln}\left[\Pr(y_i;\theta)\right]$$
  • Ici, $\Pr(y_i;\theta)$ n'est pas dfinie. Dans le cas de la rgression logistique, celle-ci peut tre remplace par $\Pr(y_i| x_i)$ vue ci-haut de faon obtenir :
$$ \begin{aligned} \max_{\beta} ~ l(\beta) & = \sum_{i=1}^n\text{ln}\left[\Lambda(x'_i\beta)^{y_i}\left(1-\Lambda(x'_i\beta)\right)^{(1 - y_i)}\right] \\ & = \sum_{i=1}^n\left(\text{ln}\left[\Lambda(x'_i\beta)^{y_i}\right] + \text{ln}\left[\left(1-\Lambda(x'_i\beta)\right)^{(1 - y_i)}\right]\right) \\ & = \sum_{i=1}^n y_i\text{ln}\left[\Lambda(x'_i\beta)\right] + \sum_{i=1}^n(1-y_i)\text{ln}\left[(1-\Lambda(x'_i\beta)\right] \end{aligned} $$
  • Pour rsoudre le problme de maximisation
$$\max_{\beta} ~ l(\beta) = \sum_{i=1}^n y_i\text{ln}\left[\Pr(y_i|x_i;\theta)\right] + \sum_{i=1}^n(1-y_i)\text{ln}\left[\Pr(y_i|x_i;\theta)\right]$$

(o l'expression $\Pr(y_i|x_i;\theta)$ a t conserve afin d'allger la notation).

  • Il suffit encore une fois de prendre la premire drive et de l'galiser zro, telle que
$$\frac{\partial l(\beta)}{\partial\beta} \Rightarrow \sum_{i=1}^ny_i\frac{\Pr'(y_i|x_i;\theta)}{\Pr(y_i|x_i;\theta)}x_i - \sum_{i=1}^n(1 - y_i)\frac{\Pr'(y_i|x_i;\theta)}{\left(1-\Pr(y_i|x_i;\theta)\right)}x_i = 0$$
  • Il est possible de montrer que si $\Pr(y_i|x_i;\theta) = \frac{\exp\{x_i'\beta\}}{1 + \exp\{x_i'\beta\}}$, alors $\Pr'(y_i|x_i;\theta) = \Pr(y_i|x_i;\theta)\left(1-\Pr(y_i|x_i;\theta)\right)$.
  • l'aide de cette galit, l'quation ci-haut peut tre rcrite telle que :
$$ \sum_{i=1}^ny_i\frac{\Pr(y_i|x_i;\theta)\left(1-\Pr(y_i|x_i;\theta)\right)}{\Pr(y_i|x_i;\theta)}x_i - \sum_{i=1}^n(1 - y_i)\frac{\Pr(y_i|x_i;\theta)\left(1-\Pr(y_i|x_i;\theta)\right)}{\left(1-\Pr(y_i|x_i;\theta)\right)}x_i = 0 \\ \sum_{i=1}^ny_i\left(1-\Pr(y_i|x_i;\theta)\right)x_i - \sum_{i=1}^n(1 - y_i)\Pr(y_i|x_i;\theta)x_i = 0 \\ \sum_{i=1}^n \left(y_i-\Pr(y_i|x_i;\theta)\right)x_i = 0 $$
  • Il n'existe pas de solution analytique cette quation. Une mthode numrique doit tre utilise pour obtenir le vecteur $\hat{\theta}$ solutionnant cette quation.
  • En remplaant l'expression $\Pr(y_i|x_i;\theta)$ par $\Lambda(x'_i\beta)$, il est possible de remarquer que $\hat{\theta}$ est alors remplac par $\hat{\beta}$.
  • Une fois le vecteur de paramtres $\hat{\beta}$ obtenu, celui-ci peut tre utilis afin d'obtenir des prdictions pour un nouveau $x_i$ de la faon suivante :
$$\hat{\pi}_i = \frac{\exp\{x_i'\hat{\beta}\}}{1 + \exp\{x_i'\hat{\beta}\}}$$

o $\hat{\pi}_i$ est la probabilit prdite par le modle que $y_i$ soit gal un pour l'observation $i$.

  • Encore une fois, cette forme l'avantage de produire des prdictions bornes entre zro et un.
  • Contrairement la rgression linaire, les coefficients estims par rgression logistique ne peuvent tre interprts directement. En effet, dans le cas de la rgression linaire :
$$E[y|X] = X\beta$$$$\Rightarrow \frac{\partial E[y|X]}{\partial x_j} = \beta_j$$
  • Dans le cas de la rgression logistique :
$$\Pr(y=1|X) = \Lambda(X\beta)$$$$\Rightarrow \frac{\partial \Pr(y=1|X)}{\partial x_j} = \Lambda'(X\beta)\beta_j$$

o la valeur de $\Lambda'(X\beta)\beta_j$ dpend de la matrice $X$.

  • Toutefois, puisque $\Lambda'(X\beta)$ est une fonction de densit, celle-ci est toujours dfinie positive. Ainsi, $\Lambda'(X\beta)\beta_j$ prend ncessairement le signe de $\beta_j$.
  • Une valeur positive de $\beta_j$ veut dire qu'une augmentation de la variable $x_j$ augmente la probabilit que la classe 1 soit observe. Une valeur ngative de $\beta_j$ signifie plutt que la probabilit que la classe 1 soit observe diminue.

Arbre de classification

  • L'arbre de classification est l'un des deux types d'arbre de dcision. Contraitrement, l'arbre de rgression, l'arbre classification prdit une classe plutt qu'une valeur.
  • L'estimation de l'arbre de classification est similaire celle de l'arbre de rgression. chaque noeud, l'algorithme cherche minimiser l'impuret des deux noeuds nouvellement crs en valuant chaque valeur de chaque variable explicative qui lui est fournie.
  • Les divisions se poursuivent jusqu' ce qu'une rgle d'arrt est atteinte. Les rgles d'arrt sont les mmes que celles pour un arbre de rgression.
  • Lorsque l'arbre est construit, celui-ci retourne pour chaque feuille la classe correspondant au mode de la variable dpendante (plutt que la moyenne dans le cas d'un arbre de rgression).
  • Comme avec le modle KNN, il est galement possible d'obtenir des probabilits plutt qu'une classe en calculant la proportion que reprsente chacune des classes dans la feuille.
  • Pour rappel, l'impuret se mesure ainsi :
$$G(Q, \theta) = \frac{N_1}{N_1 + N_2}H(Q_1(\theta)) + \frac{N_2}{N_1 + N_2}H(Q_2(\theta))$$

o $N_1$ et $N_2$ sont le nombre d'observations se retrouvant respectivement dans les noeuds potentiels 1 et 2, o $Q_1(\theta) = (y_i, x_i)~~ | ~~ x_i <= \theta$, o $Q_2(\theta) = (y_i, x_i)~~ | ~~ x_i > \theta$ et o $\theta$ est le seuil dterminer.

  • Dans le cas de la classification, l'impuret capte la distribution des classes dans un noeud. Plus les classes sont bien spares, plus chacun des noeuds crs lors de la division contient une proportion leve de seulement une des classes, et plus le noeud est pur. Cette proportion $p_{kj}$ pour une classe $k$ dans un noeud $j$ est obtenue ainsi :
$$\hat{p}_{kj} = \frac{1}{N_j}\sum_{i=1}^{N_j}1(y_i = k)$$

o $N_j$ est le nombre d'observations se trouvant dans le noeud $j$ et o $1(y_i = k)$ est une fonction indicatrice prenant la valeur un si la valeur dpendante de l'observation $i$ est gale la classe $k$ et zro autrement.

Critres de division

  • Le premier critre de division mesurant l'impuret est l'index de Gini. Il est calcul ainsi :
$$H(X_{j}) = \sum_{k=1}^K\hat{p}_{kj}(1 - \hat{p}_{kj})$$
  • Ainsi, si les classes sont parfaitement spares, $\hat{p}_{kj} = 1$ pour la classe $k$ et $\hat{p}_{-kj} = 0$ pour les autres classes. L'index de Gini est alors gal zro. Le noeud ne possde alors aucune impuret.
  • Un second critre d'impuret est l'entropie.
  • L'entropie fait rfrence la thorie de l'information. Il s'agit du niveau de dsorganisation ou d'incertitude dans un systme. Plus un systme possde une entropie leve, plus ce systme est incertain.
  • Il est ainsi possible de faire un corrolaire en probabilit. Pour une situation o il existe une probabilit qu'un vnement survienne, plus cette probabilit est leve, plus l'vnement est susceptible d'avoir lieu, moins la situation est incertaine et donc moins l'entropie de cette situation est leve.
  • Dans le cas des noeuds crs lors de l'estimation de l'arbre de classification, la mme logique s'applique. Plus un noeud est pur, plus la proportion relative de l'une des classes y est importante et plus son entropie est faible.
  • L'entropie se calcule de la faon suivante :
$$H(X_{j}) = - \sum_{k = 1}^K\hat{p}_{kj}\log \hat{p}_{kj}$$
  • Encore une fois, si les classes sont parfaitement spares, $\hat{p}_{kj} = 1$ pour la classe $k$ et $\hat{p}_{-kj} = 0$ pour les autres classes. L'entropie est alors gale zro ($\log(1) = 0$).

6.5 Classification naïve bayésienne¶

  • La classification nave baysienne (naive bayes) repose sur le thorme de Bayes :
$$\Pr(y_i = c|x_i) = \frac{\Pr(x_i|y_i = c)\Pr(y_i = c)}{\Pr(x_i)}$$

o $\Pr(y_i = c|x_i)$ est la probabilit a posteriori (posterior) que $y_i$ soit gal la classe $c$ conditionnelle $x_i$, o $\Pr(x_i|y_i = c)$ est la vraisemblance (likelihood) de $x_i$ conditionnelle ce que $y_i$ soit gal la classe $c$, o $\Pr(y_i = c)$ est la probabilit a priori (prior) que $y_i$ soit gal la classe $c$ et o $\Pr(x_i)$ est la probabilit a priori de $x_i$.

  • Ce thorme peut tre utilis pour faire de la classification. Ce thorme peut tre calcule pour chaque classe $c$. La valeur prdite est celle aossicie la probabilit la plus grande.
  • La classification nave baysienne prdit donc de la faon suivante :
$$\hat{y}_i = \underset{y}{\text{argmax}} \frac{\Pr(x_i|y_i = c)\Pr(y_i = c)}{\Pr(x_i)}$$

  • Le problme est donc simplementy d'estimer $\hat{\Pr}(x_i|y_i = c)$, $\hat{\Pr}(y_i = c)$ et $\hat{\Pr}(x_i)$.

  • Quiz : Comment faire?

  • Pour $\hat{\Pr}(y_i = c)$, il est possible d'utiliser la proportion de chacune des classes observe dans l'chantillon :
$$\hat{\Pr}(y_i = c) = \frac{1}{N}\sum_{i=1}^N 1(y_i = c)$$

o $1(y_i = c)$ est une fonction inidicatrice prenant la valeur 1 si la variable dpendante de l'inidividu $i$ provient de la classe $c$ et 0 autrement.

  • Pour $\hat{\Pr}(x_i|y_i = c)$, les choses sont plus compliques, puisque
$$\Pr(x_i|y_i = c) = \Pr(x_{1i}|x_{2i}, x_{3i}, ..., x_{ki}, y_i = c)\Pr(x_{2i}|x_{3i}, ..., x_{ki}, y_i = c)...\Pr(x_{ki}|y_i = c)$$
  • Or, les probabilits conditionnelles partielles individuelles ne sont pas connues et difficilement estimable.
  • Solution : l'indpendance des variables explicatives est suppose. De cette faon
$$ \begin{aligned} \Pr(x_i|y_i = c) & = \Pr(x_{i1}|x_{i2}, x_{i3}, ..., x_{ik}, y_i = c)\Pr(x_{i2}|x_{i3}, ..., x_{ik}, y_i = c)...\Pr(x_{ik}|y_i = c) \\ & = \Pr(x_{i1}| y_i = c)\Pr(x_{i2}|y_i = c)...\Pr(x_{ik}|y_i = c) \\ & = \prod_{j=1}^K \Pr(x_{ij}| y_i = c) \end{aligned} $$
  • Cette hypothse simplifie drlement les choses!
  • Est-elle raliste? Non! Est-ce problmatique? tonnament, pas tellement!
  • Ainsi, les probabilits des variables explicatives peuvent tre estimes indpendamment les une des autres pour chacune des classes.
  • Si $x_j$ est discrte, $\Pr(x_{ij}| y_i = c)$ est estime par la proportion empirique observe dans lchantillon :
$$\hat{\Pr}(x_{ij}| y_i = c) = \frac{1}{N_c}\sum_{i=1}^N x_i$$

o $N_c$ est le nombre d'observations ayant comme valeur de la variable dpendante la classe $c$.

  • Si $x_j$ est continue, $\Pr(x_{ij}| y_i = c)$ est modlise par une loi normale telle que
$$x_j \sim N(\bar{x}_{jc}, \sigma_{jc}^2)$$

o $\bar{x}_{jc}$ et $\sigma_{jc}^2$ sont respectivement la moyenne et la variance empirique de la variable $j$ pour la classe $c$.

  • Enfin, puisque $\Pr(x_i)$ sert uniquement normaliser les prdictions afin qu'elles sommes un, celle-ci est estime ainsi :
$$\hat{\Pr}(x_i) = \sum_{j=1}^k\hat{\Pr}(x_i|y_i = c)\hat{\Pr}(y_i = c)$$

Ainsi, la classification nave baysienne prdit donc de la faon suivante :

$$\hat{y}_i = \underset{y}{\text{argmax}} \frac{\hat{\Pr}(x_i|y_i = c)\hat{\Pr}(y_i = c)}{\sum_{j=1}^k\hat{\Pr}(x_i|y_i = c)\hat{\Pr}(y_i = c)}$$

6.6 Analyse discriminante¶

  • La classification nave baysienne repose sur l'hypothse d'indpendance des variables explicatives.
  • Hypothse forte!
  • L'analyse discrimante estime le mme problme que la classification nave baysienne, savoir
$$\Pr(y_i = c|x_i) = \frac{\Pr(x_i|y_i = c)\Pr(y_i = c)}{\Pr(x_i)}$$
  • Or, plutt que de faire l'hypothse d'indpendance des variables explicatives, $\Pr(x_i|y_i = c)$ est modlise partir d'une loi normale multivarie.
  • Ainsi, la structure suivante est suppose
$$\Pr(x_i|y_i = c) = N_p(\mu_c,\Sigma_c)$$

o $c$ reprsente chacune des classes prdire.

  • Les paramtres $\mu_k$ et $\Sigma_k$ peuvent alors tre estims simplement :
$$\hat{\mu}_c = \frac{1}{N_c}\sum_{i=1}^{N_c}x_i$$

et

$$\hat{\Sigma}_c = \frac{1}{N_c - 1}\sum_{i=1}^{N_c}(x_i-\hat{\mu}_c)(x_i-\hat{\mu}_c)'$$
  • Ensuite, comme pour la classification nave baysienne
$$\hat{\Pr}(y_i = c) = \frac{1}{N}\sum_{i=1}^N 1(y_i = c)$$

et

$$\hat{\Pr}(x_i) = \sum_{j=1}^k\hat{\Pr}(x_i|y_i = c)\hat{\Pr}(y_i = c)$$
  • L'analyse discriminante se divise en deux types : linaire et quadratique.
  • Analyse discriminante linaire (linear discriminant analysis, LDA) : La matrice $\Sigma$ est commune toute les classes.
$$\Sigma_c = \hat{\Sigma}$$
  • Analyse discriminante quadratique (quadratic discriminant analysis, QDA) : La matrice $\Sigma$ est propre chaque classe.
$$\Sigma_c = \hat{\Sigma}_c$$
  • La LDA impose une hypothse plus forte et est moins flexible.
  • En contre partie, elle est plus simple computationnellement.
