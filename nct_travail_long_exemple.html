3. Analyse exploratoire¶

Dans cette section, nous nous intressons savoir si des regroupements peuvent tre trouvs travers les diffrents combats qui se retrouvent dans notre jeu de donnes. Pour ce faire, nous utilisons la mthode des k-moyennes. Cette mthode consiste dterminer pralablement un nombre de groupes possibles puis d'assigner les observations chacun de ces groupes. Chaque groupe est reprsent par un centrode, une observation "moyenne" en quelque sorte. Les observations d'un mme groupe sont donc celles tant le plus proche d'un mme centrode. Par ailleurs, toutes les observations sont assignes l'un des centrodes.

La mthode des k-moyemmes fonctionne comme suit. Une srie de centrodes sont initialement slectionn de faon alatoire travers les observations. Celles-ci sont ensuite assignes au groupe dont le centrode est le plus prs. La proximit entre les centrodes et les observations est mesure l'aide de la distance euclidienne. Aprs la premire assignation de toutes les variables un centrode, ceux-ci sont de nouveau calculs. Leur nouvelle position travers les observations est dtermine par la position moyenne de toutes les observations qui leur sont assignes. Les centrodes ayant chang de position, les observations sont de nouveau assign au centrode le plus prs, formant ainsi de nouveaux groupes. Ce processus se repte ainsi jusqu' ce que la composition des groupes ne change plus. Les groupes sont ainsi forms d'observations partageant comme caractristique d'tre toutes plus prs du mme centrode.

Pour utiliser la distance euclidienne sur nos observations de faon adquate, celles-ci doivent d'abord tre standardises ce que nous avons dj fait. Puisque la distance euclidienne est conue pour tre utilises sur des variables continues, les variables discrtes sont omises de cette analyse explotatoire.

4. Analyse prédictive¶

Cette section prsente l'estimation et l'valuation d'un modle prdictif permettant de dterminant le vainqueur dans un affrontement entre deux Pokemon.

Le modle prdictif estim est un modle de fort alatoire de classification (en anglais, random forest). Ce modle est un modle d'ensembles puisqu'il effectue l'agrgation des rsultats provenant de diffrents modles prdictifs afin d'obtenir des prdictions plus robustes. Dans le cas du modle de fort alatoire appliqu un problme de classification, les modles agrgs sont des arbres de classification.

Un arbre de classification est un modle de prdiction non paramtrique cherchant runir des observations dans des noeuds sur la base de leur similitude dans leur variable dpendante. Ces noeuds sont obtenus la suite d'une srie de divisions binaires faites sur l'chantillon. L'arbre commence d'abord avec l'ensemble des observations composant l'chantillon dans un mme noeud. Pour effectuer sa premire division, l'algorithme produisant l'arbre de classification cherche travers l'ensemble des valeurs prises par chacune des variables explicatives celle permettant de diviser les observations de faon minimiser une mesure quelconque. Lorsque cette valeur est identifie, elle devient un critre permettant de diviser le premier noeud est deux nouveaux noeuds. Si la valeur choisie est tire d'une variable continue, la rgle de division fait en sorte que les observations possdant une valeur infrieure la valeur slectionne se retrouve dans le premier noeud alors que celles possdant une valeur suprieure se retrouve dans le second. Si la division est faite sur une variable binaire, les observations ayant pour cette variable la premire valeur se retrouve dans le premier noeud tandis que celles ayant la seconde valeur se retrouve dans le second noeud. Une fois ces noeuds crs, ceux-ci sont soumis au mme processus de division et ainsi de suite.

Il a t mentionn que les divisions sont faite de faon minimiser une mesure quelconque. Cette mesure est l'impuret des nouveaux noeuds produits. L'impuret est une mesure de la dissemblance des observations runies dans un noeud. Dans le cas de notre travail, nous entrainons un modle fort alatoire reposant sur des arbres effectuant de la classification, c'est pourquoi la mesure d'impuret utilise est l'entropie. L'entropie est une mesure de dsorganisation ou d'incertitude d'un systme. L'entropie mesure quelle point chacun des noeuds produits lors d'une division est compos d'un mlange uniforme des classes prdire. Plus la distribution des classes est uniforme dans le noeuds, plus l'entropie est leve. l'inverse, plus une classe est sureprsente par rapport une autre, plus l'entropie est faible. Suivant cette logique, l'algorithme cherche crer des noeuds sparant le mieux possible les classes.

L'algorithme produisant un arbre de classification effectue des divisions sur les noeuds jusqu' ce qu'une rgle d'arrt soient remplie. Ces rgles d'arrt sont des hyperparamtres du modle et doivent tre slectionner pour construire les arbres qui composent le modle de fort alatoire. Ces hyperparamtres dterminent des contraintes sur la dimension des arbres de faon viter le surapprentissage, c'est--dire l'incapacit du modle se gnraliser de nouvelles donnes. Dans le cas des arbres de classification, les hyperparamtres slectionner sont la profondeur maximale de l'arbre en terme de noeuds, le nombre minimal d'observations devant se retrouver dans un noeud terminal, c'est--dire un noeuds partir duquel sera fait une prdiction, et le nombre minimal d'observations devant se retrouver dans un noeud pour que celui-ci soit candidat une nouvelle division.

Les hyperparamtres sont slectionns par validation croise. La validation croise consiste diviser le groupe d'entrainement une srie de sous-groupes. Ces sous-groupes sont ensuite utiliss pour estimer le modle en omettant un des groupes de l'estimation. Ce groupe omit est ensuite utilis pour effectuer des prdictions et valuer la performance du modle. Ce processus est effectuer de faon ce que chacun des groupes soit utiliser comme groupe test. Cette procdure est ensuite effectue pour chaque ensemble de paramtres candidats possibles. Les paramtres ayant gnrs la meilleure performance sont ceux conservs pour estimer le modle sur l'ensemble des donnes.

Lorsque la construction d'un arbre est termine, de nouvelles observations peuvent lui tre fournies afin d'effectuer des prdictions pour celles-ci. Un arbre de dcision reprsentant essentiellement une srie de rgle "si/alors", l'observations n'a qu' tre classe dans un noeud terminal en suivant chacune des rgles qui s'applique elle.
