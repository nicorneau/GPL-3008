3.1 Partitionnement des données¶

  • En apprentissage automatique, l'objectif est d'entrainer un modle statistique une tche particulire.
  • La performance d'un modle se mesure gnralement par la qualit de ses prdictions.
  • Mesurer la qualit des prdictions du modle l'aide des donnes utilises pour entrainer ce modle est une mauvaise ide $\rightarrow$ Risque de surapprentissage.
  • Le surapprentissage (overfitting) survient lorsqu'un modle s'ajuste trop aux donnes qui lui sont fournies et perd ainsi sa capacit se gnraliser de nouvelles donnes.
  • Pour palier ce problme, on entraine gnralement un modle sur une partie des donnes et on mesure la qualit de ses prdictions sur une autre partie.
  • C'est ce qu'on appelle le partitionnement des donnes.

Partitionnement des donnes :

  • Gnralement en deux groupes : le groupe d'entrainement (training set) et le groupe test (test set).
  • Le groupe d'entrainement contient les donnes utilises pour faire de l'exploration et entrainer diffrents modles afin de les comparer entre eux. Ces donnes peuvent tre utilises profusion.
  • Le groupe test contient les donnes pour mesurer la qualit des prdictions d'un modle.
  • Le groupe test ne peut tre utilis qu'une seule fois $\rightarrow$ Sinon, risque de surapprentissage sur le groupe test.
  • Parfois, un troisime groupe est ajout : le groupe de validation.
  • Ce groupe sert gnralement discriminer diffrents modles prdictifs. Le meilleur modle est ensuite utilis sur le groupe test pour mesurer sa relle performance.
  • Le partitionnement est l'une des premires choses qui doit tre faite aprs avoir obtenu des donnes.
  • Il peut se faire de diffrentes faons.
  • Plus commune : chantillonage alatoire parmi les donnes disponibles.
  • Il n'y a pas de rgles pour la proportion des donnes devant tre assigne chacun des groupes (beaucoup de donnes : 70/30 60/40 pour deux groupes, 60, 20, 20 pour trois groupes).
  • Dans le cas de donnes en srie temporelle : partitionnement sur un moment prcis. Entrainement du modle jusqu' un certain moment (groupe d'entrainement), puis prdiction de la suite de la srie aprs ce moment (groupe test).
  • Attention : plus le nombre d'observations disponibles dans les donnes est faible, plus une part importante des donnes doit tre incluse dans le groupe d'entrainement.

3.2 Valeur manquante¶

  • Le nettoyage et le traitement des donnes consistent assurer une prparation adquate des donnes fournies aux diffrentes techniques statistiques.
  • La plupart des techniques statistiques sont sensibles la forme qu'ont les donnes.
  • Premire tape lors du nettoyage et du traitement des donnes est la gestion des valeurs manquantes.
  • Une valeur manquante survient lorsqu'un attribut n'est pas disponible pour un certain enregistrement.
  • Peut survenir pour de multiples raisons : donnes non disponibles, refus fournir la valeur, etc.
  • La raison de la valeur manquante est importante dans la gestion de celle-ci.
  • Une valeur manquante sur les variables explicatives (inputs, features) $\rightarrow$ possibilit de faire quelque chose.
  • Une valeur manquante sur la variable dpendante (output, label) $\rightarrow$ pas recommand de faire quelque chose.

Quoi faire?

  1. Laisser tomber les observations ayant des valeurs manquantes $\rightarrow$ Pas souhaitable lorsqu'il y a peu d'enregistrements.
  2. Laisser tomber les variables ayant des valeurs manquantes $\rightarrow$ Pas souhaitable lorsque la variable contenant les valeurs manquantes est importante.
  3. Imputation $\rightarrow$ Pas simple de dterminer avec quelle valeur faire l'imputation.
  • Une faon de reprer les valeurs manquantes est d'utiliser la mthode describe propre aux dataframes de Pandas :

3.3 Valeur aberrante¶

  • Une valeur aberrante (outliers) est une valeur qui s'carte de faon marque de la distribution de l'attribut auquel elle appartient.
  • En pratique, une valeur aberrante est problmatique si son inclusion dans le modle modifie passablement les rsultats obtenus $\rightarrow$ C'est donc davantage un problme survenant en prsence de petits chantillons.
  • Attention, une valeur aberrante peut aussi tre une valeur impossible. Ex.: un poids ngatif.
  • Parfois : signal l'analyste (ge = 999 : refus de rpondre) $\rightarrow$ Important de lire la documentation sur les donnes (s'il y en a!) et de travailler en collaboration avec les gens les connaissant bien.
  • Souvent : erreur de saisie.
  • Quoi faire? Mme chose qu'une valeur manquante + simplement ignorer la valeur aberrante!

  • Un modle statistique dans lequel la variance serait un facteur dterminant pourrait donc potentiellement mener des rsultats diffrents avec ces deux ensembles de donnes qui reprsentent pourtant la mme information.
  • Une autre situation o les chelles de mesure peuvent tre problmatiques, c'est lors de l'utilisation des mthodes calculant une mesure de distance (e.g. mthode des k-moyennes $\rightarrow$ prochain module).
  • Comment comparer la distance entre 2 ges (30 ans versus 45 ans) et deux tailles par exemple (130 lbs versus 145 lbs)? Les deux exemples diffrent de 15 units, mais ces units n'ont videmment pas la mme valeur dans les deux cas.

Quoi faire?

Retirer l'impact que peut avoir l'chelle de mesure.

  1. Normalisation
  2. Standardisation

Normalisation

  • La normalisation transforme la variable continue pour s'assurer que son support soit born entre 0 et 1.
  • Certaines mthodes statistiques ncessitent ce type de contraintes (e.g. certains rseaux de neurones)
  • Procdure :
$$\tilde{x}_i = \frac{x_i - \min(x)}{\max(x) - \min(x)}$$

poids_kg_norm = [(poids - min(poids_kg)) / (max(poids_kg) - min(poids_kg)) for poids in poids_kg]
poids_lbs_norm = [(poids - min(poids_lbs)) / (max(poids_lbs) - min(poids_lbs)) for poids in poids_lbs]

print("Poids en kilogrammes normaliss : ", np.round(poids_kg_norm, 3), "\n",

Standardisation

  • La standardisation consiste transformer les donnes afin quelles aquirent les proprits de la loi normale centre et rduite : $N(0,1)$.
  • Pour ce faire, chaque observation d'une variable est soustraite par la moyenne de cette variable puis divise par son cart type :
$$\tilde{x_i} = \frac{x_i - \bar{x}}{s}$$

o $\bar{x} = \frac{1}{N}\sum_{i=1}^N x_i$ et $s = \sqrt{\frac{1}{N-1}\sum_{i=1}^N(x_i - \bar{x})^2}$

  • La standardisation est moins sensible aux valeurs aberrantes, mais ne permet pas de contraindre le support d'une variable dans un certain intervalle.
